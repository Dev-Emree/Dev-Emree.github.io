## 2024-05-18 - Prevented Layout Thrashing in Three.js Render Loop
**Learning:** Found synchronous layout property reads (`document.body.getBoundingClientRect().top`, `window.innerWidth`, `document.body.scrollHeight`) occurring directly inside the `requestAnimationFrame` loop and `mousemove` listeners for the Three.js background canvas. This is a severe anti-pattern that forces the browser to recalculate layout every frame, causing layout thrashing and degrading GPU performance.
**Action:** Introduced variables (`cachedScrollY`, `cachedInnerWidth`, etc.) to store these layout properties. Added `passive: true` event listeners for `scroll` and `resize` to update the cached values efficiently. Replaced all direct DOM reads within the `animate()` function and `mousemove` listener with these cached properties, freeing the render loop from synchronous reflow costs.
